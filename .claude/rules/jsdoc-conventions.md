# JSDoc 規約（TypeScript strict mode）

## 基本方針

TypeScript の型で伝わる情報は JSDoc に重複させない。
JSDoc の役割は「なぜそうするか」「どんな制約があるか」の補足に限定する。

## 必須ケース

| 対象 | 書くべき内容 |
|---|---|
| `hooks/` 直下のカスタムフック | フックの目的・副作用・エラー方針 |
| ユーティリティ関数（引数が 3 つ以上、またはドメイン知識が必要） | 引数の意味と制約、戻り値の意味 |
| ドメイン定数（値の範囲や業務ルールが暗黙になっている場合） | 値の意味と制約 |

## 省略可能なケース

| 対象 | 理由 |
|---|---|
| React コンポーネント本体 | ファイル名・ディレクトリ構成から役割が明確 |
| Props 型定義の各プロパティ | 型名で意味が伝わる場合はインラインコメントで足りる |
| イベントハンドラ（`handle` プレフィックス） | 命名と引数型で自明 |
| shadcn/ui の再エクスポートコンポーネント | 追加ロジックがない場合 |
| Wayfinder 自動生成ファイル | 編集しないため不要 |
