## 概要

`.github/ISSUE_TEMPLATE/feature_request.md` のテンプレートを使って GitHub issue を作成する。
会話の流れから内容を推論し、不足している項目だけをユーザーに確認する。

## 手順

### 1. テンプレートを読む

`.github/ISSUE_TEMPLATE/feature_request.md` を読み、セクション構造を把握する。
テンプレートのセクション名・順序は変えずにそのまま使う。

### 2. 既存ラベルを取得する

```bash
gh label list --json name,description --limit 100
```

取得したラベル一覧を把握し、次のステップでの選択に備える。

### 3. コンテキストから内容を推論する

現在の会話の流れを振り返り、テンプレートの各セクションと以下を埋められるか確認する：

- **タイトル**: issue の件名（例: `ユーザープロフィール編集機能の追加`）

会話から推論できる項目は自動で埋める。ユーザーに確認なしに作成まで進めてよい。

### 4. ラベルを選択する

ステップ2で取得したラベル一覧を、issue のタイトル・概要・機能要件・制約と照合し、内容に合うラベルを選ぶ。
該当するラベルがない場合はラベルなしとする（新しいラベルは作成しない）。

### 5. 不足項目を確認する

推論できなかった項目がある場合のみ、まとめて1回質問する。項目ごとに複数回聞かない。

例（タイトルと制約が不明な場合）:
> タイトルと制約を教えてください。
> - タイトル: （issueの件名）
> - 制約: （スコープ外・技術的制約など。なければ「なし」）

### 6. issue 本文を組み立てる

テンプレートのセクションに内容を埋める。箇条書きの項目がない場合は `- なし` とする。

### 7. issue を作成する

ラベルがある場合:
```bash
gh issue create --title "[タイトル]" --body "[本文]" --assignee @me --label "label1,label2"
```

ラベルがない場合:
```bash
gh issue create --title "[タイトル]" --body "[本文]" --assignee @me
```

作成後、返ってきた issue URL をユーザーに伝える。

## 出力例

```
issue を作成しました: https://github.com/owner/repo/issues/42
```

## 注意

- `gh` コマンドが使えない場合はエラーをそのままユーザーに伝える
- テンプレートの構造は変えない（セクション名・順序を保つ）
- 内容の推論に自信がない場合は「〜と解釈しましたが合っていますか？」と一言確認してから作成する
